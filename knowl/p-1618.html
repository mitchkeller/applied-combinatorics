Even once we see that the above process does in fact generate a chain partition, it is not immediately clear that it’s a minimum chain partition. For this, we need to find an antichain of as many points as there are chains in our partition. (In the example we’ve been using, we need to find a three-element antichain.) This is where tracking the labeled vertices comes in handy. Suppose we have determined a chain \(C=\{x_1\lt x_2\lt \cdots \lt x_k\}\) using the network flow. Since \(x_1\) is the minimal element of this chain, there is no flow into \(x_1''\) and hence no flow out of \(x_1''\text{.}\) Since \(T\) is unlabeled, this must mean that \(x_1''\) is unlabeled. Similarly, \(x_k\) is the maximal element of \(C\text{,}\) so there is no flow out of \(x_k'\text{.}\) Thus, \(x_k'\) is labeled. Now considering the sequence of vertices
\begin{equation*} x_k',x_k'',x_{k-1}',x_{k-1}'',\dots,x_2',x_2'',x_1',x_1'', \end{equation*}
there must be a place where the vertices switch from being labeled to unlabeled. This must happen with \(x_i'\) labeled and \(x_i''\) unlabeled. To see why, suppose that \(x_i'\) and \(x_i''\) are both unlabeled while \(x_{i+1}'\) and \(x_{i+1}''\) are both labeled. Because \(x_i\) and \(x_{i+1}\) are consecutive in \(C\text{,}\) there is flow on \((x_i',x_{i+1}'')\text{.}\) Therefore, when scanning from \(x_{i+1}''\text{,}\) the vertex \(x_i'\) would be labeled. For each chain of the chain partition, we then take the first element \(y\) for which \(y'\) is labeled and \(y''\) is unlabeled to form an antichain \(A=\{y_1,\dots,y_w\}\text{.}\) To see that \(A\) is an antichain, notice that if \(y_i\lt y_j\text{,}\) then \((y_i',y_j'')\) is an edge in the network. Therefore, the scan from \(y_i'\) would label \(y_j''\text{.}\) Using this process, we find that a maximum antichain in our example is \(\{x_1,x_5,x_8\}\text{.}\)
